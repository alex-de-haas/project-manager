# Blockers Feature

## Overview

Blockers help teams make blocked work visible without changing the task itself. A blocker records what is stopping progress, how severe the issue is, and whether it has been resolved.

## Capabilities

- Add one or more blockers to a task.
- Assign a severity level: low, medium, high, or critical.
- Highlight blocked tasks in the task list based on the highest active severity.
- Resolve blockers with an optional resolution note.
- Reopen resolved blockers when an issue becomes active again.
- Delete blockers that are no longer relevant.
- Keep active and resolved blockers separated in the blocker dialog.

## Task List Behavior

Tasks with active blockers are visually emphasized so they are easy to identify while scanning the work list. When a task has multiple active blockers, the strongest severity controls the task highlight.

The task row also shows whether blockers exist and how many active blockers are attached to the task.

## Typical Workflow

1. Open the blocker dialog from a task row.
2. Add a short comment describing the blocking issue.
3. Choose the severity.
4. Resolve the blocker when the issue is cleared.
5. Add a resolution note when the outcome needs to be visible later.

## Severity Guidance

- **Low**: a minor impediment that does not materially change delivery expectations.
- **Medium**: a visible impediment that needs attention but has a workaround.
- **High**: a serious issue that can delay the task or related work.
- **Critical**: a blocking issue that requires immediate attention before work can continue.

## Relationship to Tasks

Blockers are attached to individual tasks. They provide delivery context and prioritization signals, while the task itself remains the source of ownership, status, checklist, and time tracking information.
